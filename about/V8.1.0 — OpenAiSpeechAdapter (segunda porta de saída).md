# V8.1.0 — OpenAiSpeechAdapter (segunda porta de saída)

## Visão Geral

Esta etapa introduz uma segunda implementação da porta `SpeechToTextPort` utilizando Spring AI integrado ao modelo Whisper-1 da OpenAI.

O objetivo da implementação é permitir alternância transparente entre transcrição local e cloud sem modificar domínio, use cases ou regras de negócio.

A arquitetura hexagonal permanece preservada através da substituição de adapters de infraestrutura orientada por configuração.

---

# O que foi implementado

## OpenAiSpeechAdapter

Foi criado adapter responsável pela integração com OpenAI Whisper via Spring AI.

### Responsabilidades

| Responsabilidade | Objetivo |
|---|---|
| Enviar áudio para OpenAI | Speech-to-text cloud |
| Implementar SpeechToTextPort | Preservar contrato |
| Isolar infraestrutura IA | Baixo acoplamento |
| Permitir troca transparente | Estratégia plugável |

---

## Seleção dinâmica de provider

A ativação do adapter ocorre via:

```java
@ConditionalOnProperty(
    name = "speech.provider",
    havingValue = "openai"
)
```

---

## Adapter padrão local

O `SpeachesAdapter` permanece como implementação padrão local.

### Estratégia

| Provider | Objetivo |
|---|---|

# Decisão de arquitetura

## Por que segunda implementação da porta?

A arquitetura hexagonal permite múltiplos adapters sem alterar regras de negócio.

Isso possibilita:

- fallback futuro
- multi-provider
- testes simplificados
- troca transparente

---

## Por que Spring AI?

Spring AI fornece:

- abstração unificada
- integração Spring Boot
- menor boilerplate
- compatibilidade futura

---

## Trade-offs adotados

| Decisão | Benefício | Trade-off |
|---|---|---|
| OpenAI cloud | Sem infraestrutura GPU | Custo variável |
| Adapter separado | Baixo acoplamento | Mais configuração |
| ConditionalOnProperty | Flexibilidade | Profiles adicionais |
| Spring AI | Integração simples | Dependência framework |

# Troubleshooting

## Erro: OPENAI_API_KEY ausente

### Causa

Variável não configurada.

### Solução

```env
OPENAI_API_KEY=sk-...
```

---

## Erro: Adapter incorreto carregado

### Causa

Property inválida.

### Solução

Validar:

```yaml
speech.provider=openai
```

---

## Erro: Timeout OpenAI

### Causa

Latência externa.

### Solução

Configurar timeout HTTP apropriado.
