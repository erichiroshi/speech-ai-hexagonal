# V6.1.0 — Micrometer + Prometheus

## Visão Geral

Esta etapa introduz a camada inicial de observabilidade da aplicação utilizando Micrometer e Prometheus.

O objetivo é centralizar métricas técnicas e de negócio relacionadas ao fluxo de transcrição permitindo monitoramento operacional, análise de performance e preparação para dashboards e tracing distribuído.

---

# O que foi implementado

## Micrometer

Foi adicionada a dependência:

```gradle
implementation 'io.micrometer:micrometer-registry-prometheus'
```

---

## TranscriptionMetrics

Foi criado componente central responsável pelas métricas da aplicação.

### Métricas adicionadas

| Métrica | Tipo | Objetivo |
|---|---|---|
| transcription.requests.total | Counter | Total de requisições |
| transcription.requests.error | Counter | Total de erros |
| transcription.speaches.duration | Timer | Latência Whisper |
| transcription.file.size.bytes | DistributionSummary | Tamanho de arquivos |

---

## Percentis de latência

O Timer publica:

- p50
- p95
- p99

```java
Timer.builder("transcription.speaches.duration")
  .publishPercentiles(0.5,0.95,0.99)
  .register(registry);
```

---

## Prometheus

Foi adicionado container Prometheus ao ambiente Docker.

### Objetivos

- Scraping de métricas
- Persistência temporal
- Integração futura com Grafana

# Decisão de arquitetura

## Por que Micrometer?

Micrometer abstrai backend de métricas permitindo:

- Prometheus
- Datadog
- New Relic
- Grafana Cloud

sem alterar código de instrumentação.

---

## Por que centralizar métricas?

Evita:

- métricas duplicadas
- nomes inconsistentes
- tags divergentes
- espalhamento operacional
