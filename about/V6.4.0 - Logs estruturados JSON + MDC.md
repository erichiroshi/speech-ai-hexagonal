# V6.4.0 — Logs estruturados JSON + MDC

## Visão Geral

Esta etapa introduz logging estruturado em JSON juntamente da propagação de contexto via MDC.

O objetivo é melhorar observabilidade operacional, correlação de eventos e integração futura com stacks centralizadas de logging.

---

# O que foi implementado

## Logback estruturado

Foi configurado:

```text
logstash-logback-encoder
```

---

## Estratégia por profile

### dev

- logs coloridos
- leitura humana

### prod

- JSON estruturado
- integração observabilidade

---

## MDC

Foi criado `MdcLoggingFilter`.

### Campos adicionados

| Campo | Objetivo |
|---|---|
| requestId | Correlação |
| traceId | Tracing |
| spanId | Tracing |
| httpMethod | Contexto |
| uri | Requisição |
| fileName | Arquivo |
| fileSizeBytes | Observabilidade |

---

## Header X-Request-Id

Todas respostas agora retornam:

```http
X-Request-Id
```

| Arquivo | Tipo | Objetivo |
|---|---|---|
| logback-spring.xml | Alterado | Configuração logs |

---

# Decisão de arquitetura

## Por que logs JSON?

JSON facilita:

- ingestão ELK
- Loki
- Datadog
- parsing automatizado

---

## Por que MDC?

Permite correlação consistente entre:

- logs
- métricas
- traces

---

# Resultado da etapa

A aplicação agora possui:

- Logs estruturados JSON
- MDC propagado
- Correlação distribuída
- RequestId automático
- Observabilidade operacional avançada
