# V4.3.0 — Testes de integração com Testcontainers

## Visão Geral

Esta etapa introduz testes de integração reais utilizando Redis via Testcontainers.

O objetivo é validar o comportamento completo do cache distribuído em ambiente isolado e reproduzível sem dependência de infraestrutura manual.

---

# O que foi implementado

## RedisContainer

Foi configurado container Redis para testes automatizados.

```java
@Testcontainers
class RedisCacheAdapterIT {

    @Container
    static RedisContainer redis =
        new RedisContainer("redis:7");
}
```

---

## RedisCacheAdapterIT

Valida:

- save
- findByHash
- TTL
- serialização
- integração Redis real

---

## RedisServiceIT

Executa fluxo ponta-a-ponta utilizando:

- contexto Spring completo
- Redis real
- hash SHA-256
- integração real de cache

---

## Objetivos dos testes

| Objetivo | Impacto |
|---|---|
| Validar integração real | Reduz regressões |
| Garantir serialização | Consistência |
| Testar TTL | Comportamento previsível |

# Decisão de arquitetura

## Por que Testcontainers?

Testcontainers fornece:

- Ambiente real
- Isolamento
- Reprodutibilidade
- Independência local

---

## Por que não mockar Redis?

Mocks não validam:

- serialização real
- TTL real
- integração real
- comportamento da infraestrutura

---

# Resultado da etapa

A aplicação agora possui:

- Testes reais com Redis
- Integração ponta-a-ponta
- Ambiente reproduzível
- Cobertura de cache distribuído
- Maior confiabilidade operacional
