# Fase 2 Item 3 — Thresholds de cobertura

## O que foi implementado

- Threshold mínimo de 60%
- Exclusão de pacotes sem regra de negócio
- Validação automática de cobertura

## Arquivos criados/alterados

- build.gradle
- README.md
- docs/quality.html

## Decisão de arquitetura

A cobertura foi aplicada apenas sobre código relevante ao domínio.

Trade-off:
- menor cobertura global visual
- métricas mais honestas

## Como testar

```bash
./gradlew jacocoTestCoverageVerification
```

## Troubleshooting
Build falhando por cobertura

Adicionar testes unitários para:
```bash
use cases
adapters
serviços de domínio
```
