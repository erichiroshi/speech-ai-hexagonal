# V8.2.0 — Testes + config de ambiente

## Visão Geral

Esta etapa introduz testes automatizados do adapter OpenAI juntamente da configuração completa de ambientes e profiles específicos para execução cloud.

O objetivo é validar integração HTTP sem dependência real da API OpenAI e formalizar configuração operacional para diferentes estratégias de execução.

# Decisão de arquitetura

## Por que MockWebServer?

Evita:

- custo real
- instabilidade externa
- rate limits
- dependência internet

---

## Por que separar profiles?

Permite ambientes independentes e alternância operacional simples.

---

## Trade-offs adotados

| Decisão | Benefício | Trade-off |
|---|---|---|
| MockWebServer | Testes determinísticos | Mais setup |
| Profile openai | Isolamento | Mais arquivos |
| API Key externa | Segurança | Configuração adicional |
| Multi-provider | Flexibilidade | Complexidade operacional |

# Troubleshooting

## Erro: 401 Unauthorized

### Causa

API Key inválida.

### Solução

Validar:

```env
OPENAI_API_KEY
```

---

## Erro: Connection refused

### Causa

Profile incorreto.

### Solução

Validar profiles ativos.

---

## Erro: MockWebServer não inicializa

### Causa

Porta ocupada.

### Solução

Executar testes isoladamente.
