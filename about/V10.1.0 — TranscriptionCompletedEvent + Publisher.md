# V10.1.0 — TranscriptionCompletedEvent + Publisher

## Visão Geral

Esta etapa introduz comunicação assíncrona baseada em eventos através do RabbitMQ.

O objetivo é desacoplar ações que ocorrem após uma transcrição bem-sucedida, permitindo que novos comportamentos sejam adicionados futuramente sem modificar o fluxo principal de transcrição.

A arquitetura hexagonal é preservada através da criação de uma nova porta de saída responsável pela publicação de eventos.

---

# O que foi implementado

## EventPublisherPort

Foi criada uma nova porta de saída responsável pela publicação de eventos de domínio.

```java
public interface EventPublisherPort {

    void publish(TranscriptionCompletedEvent event);

}
```

---

## TranscriptionCompletedEvent

Evento responsável por representar uma transcrição concluída com sucesso.

### Dados transportados

| Campo | Objetivo |
|---------|---------|
| audioHash | Identificação da transcrição |
| transcription | Conteúdo transcrito |
| filename | Arquivo original |

---

## RabbitMqEventPublisher

Implementação da porta `EventPublisherPort` utilizando RabbitMQ.

### Responsabilidades

- Serializar evento
- Publicar mensagem
- Isolar infraestrutura AMQP
- Preservar independência do domínio

# Decisão de arquitetura

## Por que eventos?

Permitem desacoplamento entre o fluxo principal e comportamentos secundários.

Sem eventos:

```text
TranscriptionService
 ├── Notificação
 ├── Auditoria
 ├── Analytics
 └── Logs
```

Com eventos:

```text
TranscriptionService
        ↓
 TranscriptionCompletedEvent
        ↓
     Consumers
```

---

## Por que RabbitMQ?

RabbitMQ fornece:

- entrega confiável
- filas duráveis
- roteamento
- suporte a DLQ

---

## Trade-offs adotados

| Decisão | Benefício | Trade-off |
|----------|----------|----------|
| RabbitMQ | Desacoplamento | Infraestrutura adicional |
| EventPublisherPort | Baixo acoplamento | Mais abstração |
| Evento dedicado | Evolução segura | Mais objetos |
| Comunicação assíncrona | Escalabilidade | Complexidade operacional |

# Troubleshooting

## Erro: Connection refused

### Causa

RabbitMQ indisponível.

### Solução

Validar container ativo.

---

## Erro: Evento não publicado

### Causa

Binding ou exchange incorretos.

### Solução

Verificar configuração AMQP.

---

## Erro: Serialização da mensagem

### Causa

Payload incompatível.

### Solução

Validar conversor JSON configurado.
