# V3.6.0 — Adapter JPA + Repository PostgreSQL

## Visão Geral

Esta etapa introduz a implementação concreta da persistência utilizando Spring Data JPA e PostgreSQL.

Foi criado um adapter de saída responsável por implementar a porta `TranscriptionGateway`, mantendo o domínio desacoplado da infraestrutura ORM.

A modelagem utiliza entidade JPA separada da entidade de domínio para preservar isolamento arquitetural.

---

# O que foi implementado

## Adapter de persistência

Foi criado o adapter responsável pela comunicação entre domínio e PostgreSQL.

### Responsabilidades

| Componente | Objetivo |
|---|---|
| JPA Adapter | Implementar gateway |
| Repository Spring | Persistência ORM |
| Entity JPA | Representação relacional |
| Mapper | Conversão domínio ↔ JPA |

---

## Entidade JPA

A infraestrutura utiliza entidade específica para persistência.

```java
@Entity
@Table(name = "transcriptions")
public class TranscriptionEntity {

    @Id
    UUID id;

    String audioHash;
}
```

---

## Busca indexada por hash

Foi implementada busca otimizada utilizando `audioHash`.

### Objetivos

- Reutilização futura
- Deduplicação
- Consulta rápida
- Base para cache

---

## Constraint de unicidade

O campo `audioHash` foi configurado como único.

### Benefícios

| Benefício | Objetivo |
|---|---|
| Integridade | Evitar duplicação |
| Consistência | Mesmo áudio → mesma transcrição |
| Performance | Busca indexada |
| Deduplicação | Reuso confiável |

# Decisão de arquitetura

## Por que separar entidade JPA da entidade de domínio?

A entidade JPA contém preocupações técnicas de persistência.

Separar evita:

- Acoplamento ORM
- Dependência Hibernate no domínio
- Vazamento de annotations
- Contaminação arquitetural

---

## Por que usar adapter ao invés de repository direto?

O adapter preserva a fronteira da arquitetura hexagonal.

```text
Application
    ↓
Port SPI
    ↓
JPA Adapter
    ↓
Spring Repository
    ↓
PostgreSQL
```

---

## Trade-offs adotados

| Decisão | Benefício | Trade-off |
|---|---|---|
| Entity separada | Isolamento arquitetural | Conversão adicional |
| Adapter dedicado | Baixo acoplamento | Mais classes |
| Unique hash | Consistência | Restrição rígida |
| Spring Data JPA | Produtividade | Complexidade ORM |

# Troubleshooting

## Erro: duplicate key value violates unique constraint

### Causa

Mesmo `audioHash` sendo persistido novamente.

### Solução

Validar fluxo de deduplicação antes do save.

---

## Erro: Entity não encontrada

### Causa

Pacote JPA fora do scan do Spring.

### Solução

Garantir localização dentro do package base.
