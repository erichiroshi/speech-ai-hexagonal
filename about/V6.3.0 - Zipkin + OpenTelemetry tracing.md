# V6.3.0 — Zipkin + OpenTelemetry tracing

## Visão Geral

Esta etapa introduz tracing distribuído utilizando OpenTelemetry e Zipkin.

O objetivo é rastrear o fluxo completo das requisições através dos componentes da aplicação facilitando análise de latência, debugging distribuído e investigação operacional.

---

# O que foi implementado

## Dependências OTel

Dependências adicionadas:

```gradle
implementation 'io.micrometer:micrometer-tracing-bridge-otel'
implementation 'io.opentelemetry:opentelemetry-exporter-zipkin'
```

---

## Propagação de trace

Fluxo rastreado:

```text
Controller
  ↓
Service
  ↓
SpeachesAdapter
```

---

## MDC automático

Os seguintes campos agora são propagados automaticamente:

- traceId
- spanId

---

## Configuração Zipkin

```yaml
management.tracing.sampling.probability=1.0
management.zipkin.tracing.endpoint=http://zipkin:9411/api/v2/spans
```

---

## Container Zipkin

Foi adicionado Zipkin ao ambiente Docker.

# Decisão de arquitetura

## Por que OpenTelemetry?

OTel fornece padrão aberto compatível com múltiplos vendors.

---

## Por que tracing distribuído?

Permite:

- análise ponta-a-ponta
- debugging de latência
- investigação de falhas
- correlação operacional

---

# Resultado da etapa

A aplicação agora possui:

- Tracing distribuído
- Integração Zipkin
- Propagação de spans
- MDC automático
- Observabilidade ponta-a-ponta
