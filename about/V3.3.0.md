# V3.3.0 — Serviço de Geração de AudioHash

## Visão Geral

Esta etapa introduz o serviço responsável pela geração determinística do `audioHash` utilizando SHA-256 hexadecimal baseado no conteúdo binário do áudio.

O objetivo desta implementação é centralizar a lógica de geração de hash em um componente reutilizável da camada de aplicação, garantindo consistência futura na deduplicação de transcrições e evitando dependência direta de APIs criptográficas dentro do domínio.

A implementação mantém a arquitetura hexagonal preservada, isolando responsabilidades e evitando acoplamento com infraestrutura externa.

---

# O que foi implementado

## Serviço de geração de hash

Foi criado um serviço dedicado exclusivamente à geração do hash SHA-256 do áudio recebido pela aplicação.

O componente recebe o conteúdo binário do áudio (`byte[]`) e retorna uma representação hexadecimal determinística.

### Responsabilidades

| Responsabilidade | Objetivo |
|---|---|
| Gerar SHA-256 | Identificação única do conteúdo |
| Garantir determinismo | Mesmo áudio → mesmo hash |
| Encapsular API criptográfica | Evitar acoplamento |
| Centralizar lógica | Reutilização entre use cases |

---

## Estratégia utilizada

A implementação utiliza `MessageDigest` da JDK para geração do SHA-256.

### Fluxo interno

```java
MessageDigest md = MessageDigest.getInstance("SHA-256");

byte[] hash = md.digest(audioBytes);

return HexFormat.of().formatHex(hash);
```

# Decisão de arquitetura

## Por que criar um serviço dedicado?

Centralizar a responsabilidade evita:

- Duplicação de código
- Estratégias inconsistentes
- Dependência espalhada da JDK criptográfica
- Acoplamento técnico nos use cases

---

## Trade-offs adotados

| Decisão | Benefício | Trade-off |
|---|---|---|
| Serviço isolado | Reutilização | Mais abstração |
| SHA-256 | Baixa colisão | Custo computacional |
| Hexadecimal | Facilidade persistência | Tamanho maior |
| Hash determinístico | Consistência | Sensível ao conteúdo binário |

# Troubleshooting

## Erro: NoSuchAlgorithmException

### Causa

Algoritmo SHA-256 não encontrado na JVM.

### Solução

Garantir utilização da implementação padrão Java:

```java
MessageDigest.getInstance("SHA-256");
```

---

## Erro: Hash diferente para mesmo áudio

### Causa

Alteração no conteúdo binário recebido.

### Solução

Garantir leitura consistente do arquivo sem mutações intermediárias.

---

## Erro: Conversão hexadecimal inválida

### Causa

Conversão manual incorreta de bytes.

### Solução

Utilizar:

```java
HexFormat.of().formatHex(hash);
```
