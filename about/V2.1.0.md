# Fase 2 Item 1 — Infraestrutura Local do SonarQube

## O que foi implementado

- Adição do serviço SonarQube no compose.yml
- Persistência de dados e plugins via Docker volumes
- Exposição da porta 9000
- Healthcheck para validação do container
- Documentação de configuração do kernel Linux

## Arquivos criados/alterados

- compose.yml
- README.md
- CLAUDE.md
- docs/index.html
- docs/roadmap.html

## Decisão de arquitetura

O SonarQube foi isolado como serviço auxiliar de qualidade.

Trade-off:
- aumenta consumo local de memória
- simplifica validação contínua de qualidade

## Como testar

```bash
docker compose up -d
```

Acessar:
```bash
http://localhost:9000
```
Troubleshooting
Erro vm.max_map_count

```bash
sudo sysctl -w vm.max_map_count=524288
```
