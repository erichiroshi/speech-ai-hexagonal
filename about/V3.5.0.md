# V3.5.0 — GetTranscriptionByHashUseCase

## Visão Geral

Esta etapa introduz o caso de uso responsável pela recuperação de transcrições previamente persistidas utilizando o `audioHash` como identificador lógico.

O objetivo da implementação é preparar a aplicação para reutilização inteligente de transcrições antes do envio para processamento de IA, reduzindo chamadas desnecessárias ao motor Whisper e diminuindo custo computacional.

A lógica permanece totalmente desacoplada de PostgreSQL, JPA ou qualquer detalhe de infraestrutura.

---

# O que foi implementado

## Caso de uso de recuperação

Foi criado um use case dedicado exclusivamente à busca de transcrições persistidas utilizando `audioHash`.

### Responsabilidades

| Responsabilidade | Objetivo |
|---|---|
| Buscar por hash | Recuperar transcrição existente |
| Reutilização futura | Evitar retranscrição |
| Encapsular regra de consulta | Centralizar fluxo |
| Isolar infraestrutura | Preservar arquitetura |

---

## Fluxo principal

```java
Optional<Transcription> result =
    gateway.findByAudioHash(audioHash);
```

---

## Estratégia arquitetural

O caso de uso depende apenas da porta `TranscriptionGateway`.

Nenhum detalhe relacionado a:

- PostgreSQL
- Hibernate
- JPA
- SQL
- Repositories Spring

é conhecido pela camada de aplicação.

# Decisão de arquitetura

## Por que criar um use case separado?

A recuperação por hash representa uma responsabilidade distinta da persistência.

Separar os fluxos mantém:

- SRP
- Facilidade de testes
- Evolução independente
- Composição futura com cache

---

## Por que o use case não conhece PostgreSQL?

A camada de aplicação deve depender apenas de abstrações.

A infraestrutura continua encapsulada nos adapters.

---

## Trade-offs adotados

| Decisão | Benefício | Trade-off |
|---|---|---|
| Use case dedicado | Clareza arquitetural | Mais classes |
| Busca via hash | Reuso inteligente | Necessidade de índice |
| Optional | Semântica explícita | Tratamento adicional |
| Porta SPI | Baixo acoplamento | Mais abstrações |

# Troubleshooting

## Erro: Optional vazio inesperadamente

### Causa

Hash inexistente na persistência.

### Solução

Validar geração consistente do SHA-256.

---

## Erro: Dependência de repository no use case

### Causa

Acoplamento incorreto da aplicação com infraestrutura.

### Solução

O use case deve depender apenas de:

```java
TranscriptionGateway
```
