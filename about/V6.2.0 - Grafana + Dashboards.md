# V6.2.0 — Grafana + Dashboards

## Visão Geral

Esta etapa introduz visualização operacional utilizando Grafana integrado ao Prometheus.

Foi configurado provisioning automático de datasource e dashboards permitindo ambiente observável pronto para uso local ou cloud.

---

# O que foi implementado

## Container Grafana

Foi adicionado Grafana ao ambiente Docker.

### Recursos

- Provisioning automático
- Datasource Prometheus
- Dashboards versionados
- Persistência local

---

## Dashboard de transcrições

O dashboard inclui:

| Métrica | Objetivo |
|---|---|
| req/min | Throughput |
| cache hit rate | Eficiência cache |
| latência p95 | Performance |
| erros por tipo | Observabilidade |

---

## Provisioning automático

```yaml
volumes:
  - ./observability/grafana/provisioning:/etc/grafana/provisioning
  - ./observability/grafana/dashboards:/var/lib/grafana/dashboards
```

---

## Compatibilidade Grafana Cloud

Os dashboards foram estruturados para:

- exportação simples
- versionamento Git
- reutilização cloud

---

# Arquivos criados/alterados

| Arquivo | Tipo | Objetivo |
|---|---|---|
| docker-compose.yml | Alterado | Container Grafana |
| observability/grafana/provisioning/* | Criado | Datasources |
| observability/grafana/dashboards/* | Criado | Dashboards |
| dashboard-transcriptions.json | Criado | Dashboard principal |

---

# Decisão de arquitetura

## Por que provisioning automático?

Evita configuração manual repetitiva e facilita:

- onboarding
- CI local
- ambientes reproduzíveis

---

## Por que dashboards versionados?

Dashboards passam a evoluir junto do código da aplicação.
