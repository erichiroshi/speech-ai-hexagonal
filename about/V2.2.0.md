# Fase 2 Item 2 — Plugins JaCoCo e SonarQube

## O que foi implementado

- Plugin jacoco
- Plugin org.sonarqube
- Encadeamento de tasks Gradle
- Geração automática de relatórios

## Arquivos criados/alterados

- build.gradle
- README.md
- CLAUDE.md
- docs/architecture.html

## Decisão de arquitetura

A pipeline de qualidade foi integrada diretamente ao lifecycle do Gradle.

Trade-off:
- builds mais lentos
- maior confiabilidade dos relatórios

## Como testar

```bash
./gradlew test jacocoTestReport sonar
```

## Troubleshooting  
Relatório XML ausente

Executar:
```bash
./gradlew clean test jacocoTestReport
```
