# V12.1.0 — Documentação OpenAPI e Swagger

## Visão Geral

Esta etapa introduz a documentação formal da API através do padrão OpenAPI 3 utilizando SpringDoc.

O principal objetivo não é apenas disponibilizar Swagger UI, mas estabelecer uma abordagem arquitetural que mantenha a documentação desacoplada da implementação HTTP.

Para atingir esse objetivo, cada controller passa a implementar uma interface específica localizada em um pacote `documentation`, concentrando toda a documentação OpenAPI fora da classe concreta do adapter HTTP.

Essa estratégia preserva a responsabilidade única dos controllers e reduz o acoplamento da camada de infraestrutura com a ferramenta de documentação.

---

## OpenApiConfig

Exemplo simplificado:

```java
@Bean
public OpenAPI openAPI() {
    return new OpenAPI()
        .info(
            new Info()
                .title("Speech AI Hexagonal")
                .version("v1")
                .description("API para transcrição e análise de áudio")
        );
}
```

## Interfaces de documentação

Novo padrão arquitetural:

```text
http
├── controller
│   ├── TranscriptionController
│   └── documentation
│       └── TranscriptionControllerDocumentation
```

---

## Documentação desacoplada

Exemplo:

```java
public interface TranscriptionControllerDocumentation {

    @Operation(
        summary = "Transcrever áudio"
    )
    ResponseEntity<TranscriptionResponse> transcribe(
        MultipartFile file
    );

}
```

Controller:

```java
@RestController
public class TranscriptionController
    implements TranscriptionControllerDocumentation {

}
```

# Decisão de arquitetura

## Por que usar interfaces de documentação?

O controller é um adapter HTTP.

A documentação OpenAPI representa uma preocupação secundária.

Separar essas responsabilidades melhora:

* legibilidade
* manutenção
* organização
* reutilização

---

## Por que SpringDoc?

Benefícios:

* integração nativa com Spring Boot
* OpenAPI 3
* Swagger UI automático
* baixa configuração

---

## Por que configuração global?

Evita repetição de metadados em múltiplos controllers.

---

## Trade-offs

| Decisão                    | Benefício               | Trade-off             |
| -------------------------- | ----------------------- | --------------------- |
| SpringDoc                  | Documentação automática | Dependência adicional |
| Interfaces de documentação | Baixo acoplamento       | Mais arquivos         |
| Configuração centralizada  | Consistência            | Estrutura adicional   |
| OpenAPI 3                  | Padronização            | Curva de aprendizado  |

# Troubleshooting

## Swagger UI não abre

### Causa

Dependência ausente.

### Solução

Validar:

```gradle
springdoc-openapi-starter-webmvc-ui
```

---

## Endpoint não aparece

### Causa

Método sem anotação OpenAPI.

### Solução

Verificar interface documentation.

---

## OpenAPI vazio

### Causa

Controller não implementa interface documentada.

### Solução

Validar implementação da interface.
