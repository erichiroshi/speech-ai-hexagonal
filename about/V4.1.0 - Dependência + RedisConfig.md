# V4.1.0 — Dependência + RedisConfig

## Visão Geral

Esta etapa introduz a infraestrutura inicial de cache distribuído utilizando Redis.

Foi adicionada a dependência `spring-boot-starter-data-redis` juntamente da configuração centralizada do `RedisTemplate` utilizando serialização JSON baseada em Jackson.

O objetivo desta implementação é preparar a aplicação para reutilização rápida de transcrições através de cache distribuído orientado por `audioHash`.

---

# O que foi implementado

## Dependência Redis

Foi adicionada a integração oficial do Spring Data Redis.

```gradle
implementation 'org.springframework.boot:spring-boot-starter-data-redis'
```

---

## RedisTemplate

Foi configurado um `RedisTemplate<String, Transcription>` customizado.

### Objetivos

- Serialização JSON
- Compatibilidade entre ambientes
- Reutilização de objetos
- Cache tipado

---

## Serialização JSON

Foi utilizado:

```java
Jackson2JsonRedisSerializer
```

### Benefícios

| Benefício | Objetivo |
|---|---|
| Leitura humana | Debug facilitado |
| Compatibilidade | Evolução futura |
| Performance | Serialização eficiente |
| Portabilidade | Estrutura previsível |

---

## TTL configurável

O TTL padrão do cache foi externalizado via propriedades da aplicação.

### Objetivos

- Controle operacional
- Ajuste sem recompilação
- Estratégia flexível de retenção

# Decisão de arquitetura

## Por que Redis?

Redis foi escolhido por:

- Baixa latência
- Estrutura key-value
- TTL nativo
- Excelente integração Spring

---

## Por que serialização JSON?

JSON facilita:

- Observabilidade
- Compatibilidade futura
- Debug operacional

---

# Resultado da etapa

A aplicação agora possui:

- Infraestrutura Redis funcional
- RedisTemplate configurado
- Serialização JSON
